# Demo proof checklist

Use this operational checklist to create the final public evidence. It is intentionally not a narration script.

## Before recording

- [ ] Open the judge-accessible deployment and confirm `GET /health` responds.
- [ ] Submit the prepared public Flask repository and verify its results are current.
- [ ] Confirm an invalid URL returns an actionable inline error.
- [ ] Confirm downloads for `AGENTS.md` and `repo-map.md` work.
- [ ] Verify the analysis scope/partial warning is visible when applicable.

## In the public video

- [ ] Keep the video under three minutes and include audio.
- [ ] State the problem: agents and new contributors should not begin an edit without repository context.
- [ ] Show repository submission, evidence-pack progress, all four specialists, Master reconciliation, and both artifacts.
- [ ] Open one finding and point to severity, confidence, evidence location, reason, and recommendation.
- [ ] Explain that Codex accelerated implementation and validation.
- [ ] Explain GPT-5.6 accurately: it is an optional constrained priority pass over validated findings.
- [ ] Show Evidence Mode honestly; show Native Mode only after it visibly succeeds.

## Before publishing

- [ ] Upload the video publicly and copy its final URL into the README and submission handoff.
- [ ] Replace every `REPLACE_WITH_...` placeholder.
- [ ] Re-open every final link in an unauthenticated browser session.
- [ ] Do not label fallback output as GPT-native output.
