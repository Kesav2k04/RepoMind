# Three-minute judge path

Use this route when evaluating RepoMind from the repository before a live deployment is available.

1. Read the [README](../README.md) hero and **Judge access** table for the value proposition, current verified CI, and explicit external-evidence placeholders.
2. Inspect [the landing screen](images/01-home.png): the user enters a public repository and sees why agent-ready context matters.
3. Inspect [visible orchestration](images/02-orchestration.png): repository → bounded evidence pack → four specialist reviews → Master → artifacts.
4. Inspect [evidence-backed findings](images/03-findings.png): each published signal shows severity, confidence, source location, reason, and recommendation.
5. Inspect [Master reconciliation](images/04-reconciliation.png) and [generated artifacts](images/05-artifacts.png). These captures are authentic deterministic Evidence Mode output.
6. Verify the [CI workflow](../.github/workflows/ci.yml) runs backend tests and frontend lint/build.
7. Read [the native-mode boundary](../README.md#codex-and-gpt-56-boundaries): GPT-5.6 is optional, constrained to validated presentation priorities, and must not be claimed without a real native-mode record.

## What not to infer

- A clean result is not a full-repository safety verdict; RepoMind marks bounded scans as partial when limits are reached.
- The checked-in screenshots do not prove a hosted GPT-5.6 run.
- A live URL, public video, Devpost URL, and Codex session ID remain manual submission artifacts until their placeholders are replaced.
